# Аналитическая лабораторная работа №1 AWS. Варинат 11
![image](https://github.com/NorthPole0499/Cloud_Technologies/assets/76227471/77b4232a-c422-475c-9c43-ed182ca054aa)

### Цель работы:

Знакомство с облачными сервисами. Понимание уровней абстракции над инфраструктурой в облаке. Формирование понимания типов потребления сервисов в сервисной-модели. 
Сопоставление сервисов между разными провайдерами. Оценка возможностей миграции на отечественные сервисы.

### Дано:

1. Слепок данных биллинга от провайдера после небольшой обработки в виде SQL-параметров. Символ % в начале/конце означает,
   что перед/после него может стоять любой набор символов.
2. Google с документациями провайдера

### Ход работы

Необходимо импортировать файл .csv в Excel и определить соответствие каждого сервиса международного провайдера русскому сервису. 
Заполнить информацию об отечественных сервисах, сопоставляя их напротив AWS сервисам. 

Как было сказано выше, работа происходила в Excel, где оперировать данными из csv файла было достаточно удобно. К тому же, команда решила отказаться от использования модели Apptio TBM Unified Model (ATUM) и не заполнять первые четыре пустых столбца в силу четкой и ясной документации как у Amazon, так и у крупных российских облачных компаний.

Так что сначала брался продукт компании Amazon, а далее, вручную находился его аналог на российском рынке.

### Описание сервисов Amazon
В следующей таблице опишем встретившиеся во время выполенения работы сервисы компании Amazon.

| Сервис | Описание|
| --- | --- |
| Amazon Athena | Бессерверный интерактивный аналитический сервис, построенный на фреймворках с открытым исходным кодом, который позволяет анализировать петабайты данных там, где они находятся. Сервис основан на принципах работы с языком запросов SQL, а настраивать инфраструктуру или управлять ею не требуется. |
| Amazon S3 | Объектное хранилище, рассчитанное на хранение и извлечение любых объемов данных из любой точки сети. S3 – это простой сервис хранилища, который отличается самой высокой надежностью и практически неограниченной масштабируемостью при очень низких затратах. Возможности сервиса включают в себя хранение и извлечение объектов, контроль доступа, репликацию данных и многое другое. S3 имеет следующие подвиды, которые требуют достаточно широко раскрытия, поэтому будут перечислены в строках таблицы ниже. |
| Amazon S3 Standard | Является базовым облачным хранилищем общего назначения, которое содержит в себе все необходимые функции, такие как хранение, низкая задержка и высокая пропускная способность. Подходит для задач очень широкого спектра.
| Amazon S3 Intelligent-Tiering | Этот вид хранилища автоматически перемещает данные на наиболее экономичные уровни доступа, исходя из частоты получения доступа к ним, тем самым снижает стоимость хранения. Имеет три уровня данных: частый доступ, нечастый доступ и редкий доступ. Именно между ними он переносит данные для экономии.
| Amazon S3 Express One Zone | Вид хранилища с одной зоной доступа для быстрого и стабильного доступа к регулярно используемым данным. Например, повышена скорость доступа к данным в 10 раз и снижает затраты на запросы на 50% по сравнению с S3 Standard.
| Amazon S3 Standard - Infrequent Access | Вид, предназначенный для хранения данных, доступ к которым запрашивается относительно редко. При этом сам скорость доступа остается высокой. Похоже на Standard, но, соответсвенно, с более низкими ценами. Подходит для резервного копирования или восстановления удаленных файлов.
|Amazon S3 One Zone - Infrequent Access | Вид хранилища, похожий на S3 Standard-IA. Но он отличается тем, что данные хранятся на одном уровне доступа и стоимость на 20% ниже. Вид подходит для создания экономичного решения для хранения редко используемых данных и которым не нужна доступность и устойчивость. Например, для хранения вторичных резервных копий локальных данных.
|Amazon S3 Glaicer | Это вид хранилищ, имеющий собственные три подвида. Если говорит в общем, то он преназначен для архивации данных, имеет высокую производительность и гибкость при извлечении данных.
|Amazon S3 Glaicer Instant Retrieval | Это вид архивного хранилища, отличающееся минимальной стоимостью длительного хранения данных, доступ к которым требуется редко, но скорость извлечения необходимо сохранять высокую. Экономия 68% по сравнению с Standard-IA. Тем самым, часто используется для хранения архивных данных с мгновеным доступом, например, к медицинским снимками или ресурсам новостных медиа.
| Amazon S3 Glaicer Flexible Retrieval | Вид архивного хранилище, снижающий затраты на 10 %, по сравнению с S3 Glacier Instant Retrieval, для архивных данных, которые нужно получать один или два раза в год. Он прекрасно подойдет для архивных данных, к которым не требуется мгновенный доступ, но может потребоваться внезапный доступ к огромным наборам данных, не требующий дополнительных затрат на их извлечение. К примеру, аварийное восстановление резервных копий при каком-лбио сбое.
|Amazon S3 Glaicer Deep Archive| Cамый выгодный вид архивного хранилища с поддержкой долгосрочного хранения и цифровой архивации данных. Предназначен для данных, доступ к которым запрашивается пару раз в год. Его целевой аудиторией являются клиенты, хранящие наборы данных 7–10 лет или дольше для выполнения нормативных требований в сфере здравоохранения, государственном секторе и тд. |
| APN Fee | Классификация, одобренная AWS для глобального сообщества поставщиков облачных услуг и вендоров после того, как они пройдут определенные тесты. Одобренные партнеры AWS могут участвовать в предоставлении облачных сервисов и решений AWS для клиентов, в то время как AWS предлагает технологическую платформу и маркетинг, необходимые для создания, внедрения и сбыта этих решений. Для всех уровней партнерской сети AWS требуется ежегодная плата за APN в размере 2500 долларов. |
| Amazon App Stream | Cервис вычисления для конечных пользователей AWS, который можно настроить для потоковой передачи приложений SaaS или доставки виртуальных рабочих столов с выборочным сохранением данных. Он позволяет пользователям получать доступ к своим приложениям и рабочим столам из любого места, где есть доступ в интернет. AppStream также предоставляет возможность масштабирования ресурсов в зависимости от потребностей пользователя. |
| Amazon Lex | Сервис, который позволяет создавать голосовых помощников для приложений, ботов для мессенджеров и других устройств. Он использует технологии искусственного интеллекта и машинного обучения для обработки естественного языка, чтобы понимать и отвечать на запросы пользователей. Также, Amazon Lex можно не только использовать для создания голосвых помощников, но и для анализа звукового текста. |
| Amazon Lightsail | Сервис для создания и управления виртуальными серверами. Он предлагает простую и доступную альтернативу традиционным выделенным серверам, позволяя пользователям быстро и легко создавать свои собственные серверы в облаке. Lightsail также предоставляет инструменты для управления ресурсами сервера, такие как масштабирование ресурсов, мониторинг производительности и управление доступом. |
| Amazon Neptune | Бессерверная графовая база данных с превосходной масштабируемостью и доступностью. В базу данных Neptune встроены функции безопасности, непрерывного резервного копирования и интеграции с другими сервисами AWS. Также, имеется аналитическое ядро базы данных, позволяющее быстро анализировать большие объемы графовых данных для получения аналитики и выявления тенденций. |
| Amazon Managed Blockchain | Полностью управляемый сервис (блокчейн-платформа), призванный помочь создавать отказоустойчивые приложения Web3 как в публичных, так и в частных блокчейнах. Содержит инструменты и подсервисы для создания, тестирования и развертывания блокчейн-приложений. Например, сервис предоставляет общедоступные узлы блокчейнов для Ethereum, Polygon и Bitcoin, а возможность создавать частные блокчейн-сети - с помощью фреймворка Hyperledger Fabric. |
| Amazon Maice | Cервис, который обеспечивает безопасность и конфиденциальность, используя машинное обучение (ML) и сопоставление со специальными шаблонами для обнаружения и защиты конфиденциальных данных. |
| Amzon ML | Cервис, который предоставляет инструменты и технологии для создания, тренировки и развертывания моделей машинного обучения. Он включает в себя инструменты для подготовки данных, выбора модели, оценки качества модели и развертывания модели в production. С помощью Amazon ML, разработчики и data scientists могут создавать высокоточные модели машинного обучения без необходимости написания кода или настройки инфраструктуры. Имеет два ключевых подвида, которые будут перечислены и описаны в стобцах ниже.|
| Amazon ML SageMaker | Сервис для создания моделей машинного обучения для любых стандартных примеров использования. С его помощью можно создавать, обучать и развертывать модели. Более того, сервис содержит полностью управляемую инфраструктуру, интсрументы и рабочие процессы. |
| Amazon ML Bedrock | Сервис, используемый для быстрого создания и масшатбирования генеративных приложений искусственного интеллекта с помощью базовых моделей. Bedrock предлагает большой выбор высокопроизводительных моделей от ведущих компаний в сфере ИИ. | 
| AWS Config | Инструмент конфигурации, который позволяет организациям управлять, отслеживать и контролировать изменения в своих AWS-ресурсах. AWS Config обеспечивает централизованное управление конфигурациями, автоматическое обнаружение изменений и управление правилами. |
| AWS Cost Explorer | Обозреватель затрат AWS предлагает удобный интерфейс для визуализации, анализа затрат и использования ресурсов AWS, а также для управления этими параметрами в долгосрочной перспективе. Имеется возможность быстро приступать к работе с помощью пользовательских отчетов для анализа расходов и использования. |
| AWS Global Accelerator | Cервис ускоренной доставки контента, который помогает ускорить загрузку веб-сайтов и приложений для клиентов по всему миру. Он использует сеть распределенных точек присутствия (POP) AWS и технологию локального кэширования для оптимизации доставки контента клиентам, позволяя разработчикам и владельцам сайтов определять географические регионы, для которых требуется ускорение, и автоматически оптимизировать доставку контента на основе этих настроек. |
| AWS Glue | Бессерверный сервис интеграции данных, который упрощает поиск, подготовку, перемещение и интеграцию данных из множества источников для анализа, машинного обучения и разработки приложений. А именно, сервис использует Apache Spark для обработки данных и позволяет создавать ETL-процессы (извлечение, преобразование и загрузка данных). |
| AWS Greengrass | Периферийная среда выполнения и облачный сервис с открытым исходным кодом для создания, развертывания и управления ПО устройств. Платформа позволяет разработчикам создавать масштабируемые и надежные IoT-решения. Предоставляет собой набор сервисов и инструментов для управления устройствами, обработки данных, безопасности и тд. |

### Описание отечественных сервисов 
В следующей таблице опишем встретившиеся во время выполенения работы сервисы российских компании.

| Сервис | Описание|
| --- | --- |
| Yandex Query | Сервис виртуализации данных для инженеров и аналитиков, который позволяет выполнять аналитические и потоковые запросы в режиме реального времени к структурированным и частично структурированным данным с помощью YQL — диалекта SQL. С помощью аналитических запросов в Yandex Query вы сможете получать данные из Yandex Object Storage в форматах JSON, CSV, TSV, Parquet и др. |
| Yandex Object Storage | Универсальное масштабируемое облачное объектное хранилище. Объекты в Object Storage хранятся в нескольких географически распределённых зонах доступности. Также, объектное хранилище S3 автоматически расширяется по мере необходимости. В сервисе доступны следующие классы хранилищ: стандартное, холодное и ледяное. Более холодные классы предназначены для длительного хранения объектов, работать с которыми планируется реже. Чем холоднее хранилище, тем дешевле хранить в нем данные, но тем дороже их читать и записывать. Похожее есть в Amazon S3, но там отдельные класс хранилищ выделены в отедльные подвиды и имеют разбивку на разную скорость получения данных, отчего также зависит их стоимость.|
| Yandex Data Streams | Масштабируемый сервис для управления потоками данных в режиме реального времени. При использовании в качестве транспорта для микросервисов упрощает интеграцию, увеличивает надёжность, улучшает масштабирование. К тому же, один поток данных может быть передан в несколько приёмников с различными политиками хранения. |
| SberDevices Dialog Platform | Платформа для создания голосовых помощников и чат-ботов, разработанная компанией SberDevices. Она предоставляет инструменты и технологии для обработки естественного языка, распознавания речи, синтеза голоса и других функций, необходимых для создания интерактивных голосовых приложений. |
| Yandex Compute Cloud | Сервис предоставляет масштабируемые вычислительные мощности для размещения, тестирования и прототипирования всевозможных проектов. Имеется возможность разместить виртуальную машину в любой из трёх географически распределённых зон доступности. Масштабируемость либо автоматческая, либо по требованию. |
| Comindware ElasticData |  Графовая база данных, созданная российской компанией. Технология обеспечивает высокий уровень гибкости, масштабирования и скорости работы веб-ориентированных систем и приложений, созданных с её помощью. |
| QIWI Blockchain Technologies | Плтформа для разработки решений на основе блокчейн-технологии. Предлагаются типовые решения для финансовых учреждений, государственных органов и других организаций. Также, предлагаются отдельные интсрументы для тестирования и развертывания блокчейн-приложений. |
| Yandex Smart Web Security | Сервис для защиты веб-приложений от DDoS-атак на уровне приложений и ботов. Есть встроенный мониторинг и логирование, а также фильтрация трафика. Так как сервис находится только лишь на стадии Preview в нём ещё совершенствуются и развиваются технологии защиты с помощью ML-технологий. |
| Yandex DataSphere | Сервис для ML-разработки с интуитивным интерфейсом. В DataSphere есть все необходимые инструменты и динамически масштабируемые облачные ресурсы для полного цикла разработки машинного обучения. Можно не только выбирать конфигурацию вычислительных ресурсов, но и есть встроенные известные IDE. |
| Yandex Audit Trails | Сервис для сбора и выгрузки аудитных логов ресурсов Yandex Cloud. Пристусвует гибкая настройка сбора уадитных логов и оперативный мониторинг. С данным сервисом станет возможным расследование инцедентов и контроль за действиями пользователей. |
| Yandex Cloud Billing | Сервис позволяет получать информацию об объеме использованных ресурсов, проверять расходы денежных средств и оплачивать потребление ресурсов. Так как в экосистеме Yandex Cloud пользователь платит только за потребленные ресурсы и время их фактического использования, то данный инструмент станет настоящим помощником в анализе затрат и использовании ресурсов. |
| Selectel SDN | Сеть доставки контента cостоит из множества узлов, благодаря которым ускоряется раздача статического контента. Сеть кеширующих серверов позволяет решать задачи, связанные с качеством и скоростью доставки контента географически распределенным пользователям, и в том числе снижать нагрузку на основные серверы веб-проектов. У Selectel более 25 точек пристуствия только в странах СНГ. |
| Yandex IoT Core | Сервис интернета вещей для двустороннего обмена сообщениями между реестрами и устройствами по протоколу MQTT с автоматическим масштабированием. Сервис даёт инструменты для управления устройствами и организации. |

### Выводы о возможностях миграции

Amazon несомненно является мировым гигантом облачных сервисов. Не всем представленным сервисам в нашей таблице нашлись аналоги на российском рынке. Однако, хочется сказать, что у основных игроков отечественного рынка облачных технологий, то есть у Yandex, Selectel и Sber, есть преимущества, имея которые возможно тягаться за первенство с Amazon. 

К примеру, что у Amazon, что у Yandex идеально прописанная документация. Также, у компании Yandex есть аналоги почти всех основных сервисов Amazon. Единственный минус, который можно отметить, так это - недостаток времени для развития. Большое количество российских сервисов пока что слишком недоделанные или вообще находятся в стадии тестирования. Но можно с уверенностью сказать, что в ближайшие пару лет будет возможно мигрировать с сервисов Amazon, например, на сервисы того же Yandex.
